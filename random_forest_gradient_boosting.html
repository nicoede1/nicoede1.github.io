def wrangle(filename):
    
    """
    Read a JSON file into a 'DataFrame'.

    Returns treated Data in a pandas DataFrame

    Parameters
    ----------
    filename: str
        name of the file to be read
    """
    
    # Open file and load JSON
    with open(filename, "r") as f:
        data = json.load(f)
    
    # Turn dictionary into DataFrame
    df = pd.DataFrame().from_dict(data["data"]).set_index("company_id")
    
    return df

A few outtakes can be taken from the initial EDA. First, the data is imbalanced. This is something to address in the data preparation. Second, many of the features have missing values that need to be impute. And since the features are highly skewed, the best imputation strategy is likely median, not mean. Finally, there are autocorrelation issues, which means that linear models will not be efficient. A tree-based model it is for now.

Because the classes are imbalanced, the baseline accuracy is very high. It is important to keep this in mind because, even if the trained model gets a high validation accuracy score, that does not mean it is actually good.

Clearly the model with the under-sampling approach performed the worst. The other two approaches were quite similar

Good accuracy scores do not tell much about the model's performance when dealing with imbalanced data. So instead of looking at what the model got right or wrong, looking at what predictions differ for the two classes in the dataset is ideal

In [37]:
# Plot confusion matrix for the regular model

With the confusion matrix it is possible to see that although the model is predicting the True Falses very well it is not predicting True Positives that well

It is also possible to evaluate the model using precision and recall. The precision score is important when it is desirable for the model to only predict that a company will go bankrupt if its very confident in its prediction. The recall score is important if there is a need to identify all the companies that will go bankrupt, even if that means being incorrect sometimes.

Models that predict a single outcome are definitely useful but are not always the optimum option. It is definitely a useful way to predict the future, but when using more than one model simultaneously, it is possible to have a more trustworthy prediction. Ensemble models work by building multiple models on random subsets of the same data, and then comparing their predictions to make a final prediction. That is one of the reasons that a random forest may be the best option for now
